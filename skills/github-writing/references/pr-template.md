# PR Description Template

Copy and adapt. Delete sections that don't apply.

```markdown
## Summary

<!-- 1–2 sentences: what changed and why -->

## Motivation

<!-- Link to issue, explain the problem -->
Fixes #

## Changes

- <!-- Key change 1 -->
- <!-- Key change 2 -->
- <!-- Key change 3 -->

## Testing

- [ ] <!-- How this was tested -->
- [ ] <!-- What reviewers should verify -->

## Screenshots

<!-- Before/after if UI changes. Delete this section if not applicable. -->

## Notes

<!-- Breaking changes, migration steps, follow-up work, or anything reviewers should know -->
```

## Minimal PR Template

For small changes where the full template is overkill:

```markdown
<!-- One-liner: what and why -->

Fixes #
```

## Draft PR Template

For early feedback on work-in-progress:

```markdown
## WIP: <!-- Title -->

### Goal

<!-- What this PR will do when complete -->

### Current State

- [x] <!-- Done item -->
- [ ] <!-- Remaining item -->
- [ ] <!-- Remaining item -->

### Questions for Reviewers

- <!-- Specific question about approach, design, naming, etc. -->

### Not Ready For

- <!-- What reviewers should NOT review yet (e.g., tests, docs, error handling) -->
```
