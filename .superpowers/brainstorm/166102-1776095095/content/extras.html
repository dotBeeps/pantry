Psi — Additional Features for Core Shell

Click any that feel right for the MVP. These would layer onto the base layout.

A

Ally Toast Notifications

When a spawned CC or pi session reports in via sending stone, a small toast slides in from the bottom-right with the ally name and message preview. Click to jump to that session. Keeps you aware of ally activity without leaving Ember's stream.

B

Panel Drawer (Right Side)

The state panel becomes a collapsible drawer. Click a tab icon to expand it wider and show richer panels — dragon-guard permissions, kobold task board, memory vault browser. Collapsed by default so the thought stream stays wide. Foundation for sub-project 3.

C

Command Palette (Ctrl+K)

Quick fuzzy-search overlay for switching sessions, opening panels, jumping to settings, running daemon commands. Same muscle memory as VS Code / Raycast. Keeps the chrome minimal while everything stays one shortcut away.

D

Thought Stream Filters

Toggle buttons in the stream header to show/hide event types: think, speak, observe, beat. When Ember gets chatty during a long thought cycle, you can filter down to just speak events to see what she actually said vs. her inner monologue.

E

Theme System

Dark by default (as shown), but with a theme system that can be swapped. Not just light/dark — let personas influence the color accent. Ember gets warm amber, Maren gets her own palette. The active persona tints the UI.

F

Connection Health Bar

Thin strip at the top showing SSE connection status per persona — connected (green dot), reconnecting (amber pulse), disconnected (red). Since psi talks to multiple doggy ports, you need to see at a glance which daemons are alive. Auto-reconnect with backoff.
