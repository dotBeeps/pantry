How psi gets its data
■ Thought stream — doggy SSE. Real-time thoughts, speak events, observations, beat markers. Psi just renders what arrives.
■ User input — POST /message to doggy. Psi sends text, daemon enqueues as sensory event for next thought cycle.
■ Everything else — MCP JSON-RPC. Memory vault, quest status, attention, stone messages, config, guard state. Psi calls MCP tools and renders responses.
■ Harness data — CC/pi sessions connect to daemon MCP, register themselves, report via stone messages. Psi sees them through the daemon, never directly.
